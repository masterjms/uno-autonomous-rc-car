# Arduino 2WD Autonomous RC Car

Arduino Uno 기반 2휠 RC카 키트를 사용하여, 초음파 센서 하나만으로 물리적 장애물을 회피하는 자율주행 RC카 프로젝트입니다.

이 프로젝트는 라인트레이싱 방식이 아니라, 강의실에 구성된 장애물 코스에서 전방 초음파 센서로 거리를 측정하고, 장애물이 가까워지면 회전 탐색을 통해 열린 공간을 찾아 이동하는 방식으로 구현되었습니다.

---

## 1. 프로젝트 목표

본 프로젝트의 목표는 단순히 RC카를 움직이는 것이 아니라, 제한된 하드웨어 환경에서 최대한 안정적이고 빠르게 장애물 코스를 통과하는 자율주행 로직을 구현하는 것입니다.

주요 목표는 다음과 같습니다.

- Arduino Uno 기반 RC카 직접 제어
- HC-SR04 초음파 센서를 이용한 전방 거리 측정
- 라인이 없는 장애물 코스에서 자율주행
- 장애물 감지 후 열린 공간 탐색
- 정확한 각도 제어가 어려운 환경에서 상태머신 기반 회피 로직 구현
- VSCode + PlatformIO 환경에서 개발
- C/C++ 기반 Arduino 코드 직접 작성

## 3. 하드웨어 구성

### 메인 보드

- Arduino Uno SMD R3
- ATmega328P 기반
- 16MHz
- SRAM 2KB
- Flash 32KB

### 모터 드라이버

- HW-130 V0.0.4 Motor Shield
- L293D 기반 Arduino Motor Shield 계열
- AFMotor 라이브러리 사용

### 구동부

- DC 모터 2개
- 전방 좌우 바퀴 구동
- 후방은 동력 없는 캐스터 휠 1개
- 왼쪽 모터: M1
- 오른쪽 모터: M4

### 센서

- HC-SR04 초음파 센서 1개
- 전방 고정 장착
- TRIG: A0
- ECHO: A1

### 차체 크기

- 길이: 약 22cm
- 너비: 약 18cm

차체 폭이 18cm이기 때문에, 초음파 센서가 전방에 열린 공간을 감지하더라도 실제로는 바퀴나 차체 모서리가 장애물에 걸릴 가능성이 있습니다.

이를 보완하기 위해 열린 공간을 찾은 뒤 곧바로 전진하지 않고, 추가 회전을 수행하도록 설계했습니다.

## 4. 핵심 문제 정의

이 프로젝트의 가장 큰 제약은 다음과 같습니다.

1. 초음파 센서가 전방에 1개뿐입니다.
2. 좌우 방향을 동시에 볼 수 없습니다.
3. 엔코더가 없어 실제 회전 각도를 알 수 없습니다.
4. 자이로 센서가 없어 차체 방향을 알 수 없습니다.
5. 바닥 마찰, 배터리 전압, 캐스터 휠 상태에 따라 회전량이 달라집니다.
6. 초음파 센서는 차체 전체 폭이 아니라 전방 중앙 거리만 측정합니다.

따라서 “정확히 90도 회전” 같은 방식은 실제 환경에서 안정적이지 않습니다.

예를 들어 코드상으로 90도 회전을 의도하더라도 실제로는 다음과 같은 문제가 생깁니다.

- 바닥 마찰이 크면 덜 회전함
- 배터리 전압이 높으면 더 회전함
- 좌우 모터 출력 차이로 회전 중심이 틀어짐
- 후방 캐스터 휠이 끌리면서 회전량이 달라짐

그래서 본 프로젝트는 각도 기반 제어가 아니라, **거리 기반 탐색 + 상태머신 기반 회피** 방식을 사용했습니다.

## 5. 상태머신을 사용한 이유

초기에는 단순히 다음과 같은 구조를 생각할 수 있습니다.

```text
전진
→ 장애물 감지
→ 좌회전
→ 거리 확인
→ 전진
```

하지만 이런 단순 if문 구조는 실제 주행에서 문제가 많았습니다.

예상되는 문제:

- 장애물을 너무 늦게 감지함
- 회전 중 초음파 값이 순간적으로 튀면 잘못된 방향으로 전진함
- 열린 공간을 찾자마자 전진하면 차체 모서리가 걸림
- 회피 후 바로 다시 장애물을 감지해 찔끔찔끔 움직임
- 후진, 회전, 전진 타이밍이 코드 안에서 섞여 디버깅이 어려움

그래서 전체 로직을 상태머신 FSM으로 분리했습니다.

상태머신을 사용한 이유는 다음과 같습니다.

- 현재 RC카가 무엇을 하고 있는지 명확하게 구분 가능
- 각 상태마다 전환 조건을 독립적으로 관리 가능
- `delay()` 중심 코드보다 디버깅이 쉬움
- 초음파 거리값, 시간 조건, 연속 감지 횟수를 함께 고려 가능
- 회피 실패 시 fallback 동작을 구조적으로 추가 가능
- 시리얼 로그에서 현재 동작 상태를 쉽게 추적 가능

## 6. 전체 주행 전략

최종 주행 전략은 다음과 같습니다.

```text
일반 주행
→ 장애물 감지
→ 정지
→ 짧게 후진
→ 왼쪽으로 회전하며 열린 공간 탐색
→ 열린 공간이 확인되면 추가 좌회전
→ 일정 시간 전진 보장
→ 일반 주행 복귀
```

왼쪽 탐색에 실패하면 오른쪽으로 크게 회전하여 다시 열린 공간을 탐색합니다.

```text
왼쪽 탐색 실패
→ 오른쪽으로 크게 회전
→ 열린 공간 탐색
→ 열린 공간이 확인되면 추가 우회전
→ 일정 시간 전진 보장
→ 일반 주행 복귀
```

양쪽 모두 실패하면 탈출 루틴을 수행합니다.

```text
양쪽 탐색 실패
→ 길게 후진
→ 큰 우회전
→ 저속 전진
→ 일반 주행 복귀
```

## 7. 상태 정의

```cpp
enum AutoState {
  ST_CRUISE = 0,
  ST_STOP_BEFORE_BACK,
  ST_BACK,
  ST_STOP_BEFORE_LEFT,
  ST_TURN_LEFT_SEARCH,
  ST_LEFT_MARGIN,
  ST_ESCAPE_FORWARD,
  ST_TURN_RIGHT_SEARCH,
  ST_RIGHT_MARGIN,
  ST_FAIL_BACK,
  ST_FAIL_TURN,
  ST_RECOVERY_FORWARD
};
```

| 번호 | 상태 이름 | 의미 |
|---:|---|---|
| 0 | `ST_CRUISE` | 일반 주행. 거리별로 고속/중속/저속 전진 |
| 1 | `ST_STOP_BEFORE_BACK` | 장애물 감지 후 후진 전 잠깐 정지 |
| 2 | `ST_BACK` | 장애물 앞에서 짧게 후진 |
| 3 | `ST_STOP_BEFORE_LEFT` | 좌회전 탐색 전 잠깐 정지 |
| 4 | `ST_TURN_LEFT_SEARCH` | 왼쪽으로 회전하며 열린 공간 탐색 |
| 5 | `ST_LEFT_MARGIN` | 왼쪽 공간 발견 후 차체 폭 보정용 추가 좌회전 |
| 6 | `ST_ESCAPE_FORWARD` | 회피 성공 후 일정 시간 전진 보장 |
| 7 | `ST_TURN_RIGHT_SEARCH` | 왼쪽 실패 후 오른쪽으로 크게 회전하며 열린 공간 탐색 |
| 8 | `ST_RIGHT_MARGIN` | 오른쪽 공간 발견 후 차체 폭 보정용 추가 우회전 |
| 9 | `ST_FAIL_BACK` | 좌우 모두 실패 시 길게 후진 |
| 10 | `ST_FAIL_TURN` | 실패 탈출용 큰 우회전 |
| 11 | `ST_RECOVERY_FORWARD` | 탈출 후 저속 전진으로 복구 |

## 8. 상태 전환 기준

### 일반 주행

```text
거리 > 55cm        → 고속 전진
40cm ~ 55cm        → 중속 전진
24cm ~ 40cm        → 저속 전진
24cm 이하 2회 연속 → 회피 시작
13cm 이하          → 즉시 회피 시작
```

### 회피 시작

```text
정지
→ 100ms 대기
→ 380ms 후진
→ 100ms 대기
→ 왼쪽 탐색 회전
```

### 왼쪽 탐색

```text
처음 380ms 동안은 무조건 좌회전
이후 거리 55cm 이상이 2회 연속 나오면 열린 공간으로 판단
열린 공간 발견 시 180ms 추가 좌회전
이후 650ms 전진 보장
```

### 오른쪽 탐색

```text
왼쪽 탐색이 1100ms 동안 실패하면 오른쪽 탐색
처음 650ms 동안은 무조건 우회전
이후 거리 55cm 이상이 2회 연속 나오면 열린 공간으로 판단
열린 공간 발견 시 180ms 추가 우회전
이후 650ms 전진 보장
```

### 양쪽 실패

```text
오른쪽 탐색까지 실패
→ 600ms 후진
→ 1000ms 큰 우회전
→ 450ms 저속 전진
→ 일반 주행 복귀
```

## 9. 주요 튜닝값

```cpp
const int EMERGENCY_DISTANCE = 13;
const int DANGER_DISTANCE    = 24;
const int SLOW_DISTANCE      = 40;
const int CLEAR_DISTANCE     = 55;

const int HIGH_SPEED = 170;
const int MID_SPEED  = 140;
const int LOW_SPEED  = 105;
const int TURN_SPEED = 140;
const int BACK_SPEED = 125;
```

```cpp
const unsigned long BACK_MS           = 380;
const unsigned long TURN_MIN_MS       = 380;
const unsigned long TURN_LEFT_MAX_MS  = 1100;
const unsigned long TURN_RIGHT_MIN_MS = 650;
const unsigned long TURN_RIGHT_MAX_MS = 1800;
const unsigned long TURN_MARGIN_MS    = 180;
const unsigned long ESCAPE_FORWARD_MS = 650;
```

### 튜닝 기준

- 장애물에 너무 가까이 붙으면 `DANGER_DISTANCE` 증가
- 너무 빨리 회피하면 `DANGER_DISTANCE` 감소
- 차체가 모서리에 걸리면 `CLEAR_DISTANCE` 또는 `TURN_MARGIN_MS` 증가
- 회피 후 찔끔찔끔 움직이면 `ESCAPE_FORWARD_MS` 증가
- 회전이 약하면 `TURN_SPEED` 증가
- 회전 중 미끄러지면 `TURN_SPEED` 감소

## 10. 초음파 센서 처리

HC-SR04는 순간적으로 값이 튈 수 있기 때문에 한 번의 측정값만으로 판단하지 않습니다.

적용한 방식:

- 60ms마다 측정
- `pulseIn()`에 timeout 적용
- 비정상 거리값 무시
- 최근 3개 거리값의 중간값 사용
- 위험거리와 열린공간 기준을 분리하여 히스테리시스 적용

```text
DANGER_DISTANCE = 24cm
CLEAR_DISTANCE  = 55cm
```

이렇게 기준을 분리한 이유는 24cm 근처에서 장애물 감지와 전진이 반복되는 것을 막기 위해서입니다.

## 11. 디버그 로그

시리얼 모니터에서는 현재 상태와 거리값을 출력합니다.

예시:

```text
state=4 dist=21 clear=0 danger=0
state=7 dist=21 clear=0 danger=0
state=9 dist=21 clear=0 danger=0
```

해석:

- `state`: 현재 FSM 상태 번호
- `dist`: 필터링된 초음파 거리값
- `clear`: 열린 공간 연속 감지 횟수
- `danger`: 위험거리 연속 감지 횟수

예를 들어 다음 로그는 왼쪽 탐색, 오른쪽 탐색, 실패 루틴까지 진행된 상황입니다.

```text
state=4 dist=21
state=7 dist=21
state=9 dist=21
state=10 dist=21
state=11 dist=21
```

이 경우 초음파 거리가 계속 21cm로 유지되므로, 실제 차체가 움직이지 않았거나 주변이 계속 막혀 있다고 판단할 수 있습니다.

## 12. 개발 중 시행착오

### 1. PlatformIO 빌드와 업로드 구분

처음에는 `pio run`을 실행하여 빌드만 수행했습니다.

```bash
pio run
```

이 명령은 컴파일만 수행하고 Arduino에 업로드하지 않습니다.

실제 업로드는 다음 명령으로 수행해야 합니다.

```bash
pio run -t upload
```


### 2. 회전각 기반 제어의 한계

초기에는 장애물을 만나면 90도 회전시키는 방식도 고려했습니다.

하지만 실제 RC카에서는 바닥 마찰, 배터리 전압, 캐스터 휠 상태에 따라 회전량이 달라집니다.

따라서 최종적으로는 다음 방식으로 변경했습니다.

```text
정확한 각도 회전
→ 열린 공간을 찾을 때까지 회전
→ 열린 공간 확인 후 추가 회전
→ 일정 시간 전진 보장
```

### 3. 차체 폭 문제

초음파 센서는 차체 중앙 전방만 측정합니다.

따라서 센서상으로는 열린 공간이어도 차체 폭 때문에 바퀴가 장애물에 걸릴 수 있습니다.

해결 방식:

```text
CLEAR_DISTANCE를 넉넉하게 설정
열린 공간 확인 후 TURN_MARGIN_MS만큼 추가 회전
회피 후 바로 고속 전진하지 않고 중속 전진
```

## 13. 현재 한계

현재 하드웨어만으로는 다음 한계가 있습니다.

- 정확한 회전각 측정 불가
- 좌우 공간을 동시에 판단할 수 없음
- 차체 위치나 방향을 알 수 없음
- 얇은 장애물이나 비스듬한 장애물은 초음파가 놓칠 수 있음
- 배터리 전압에 따라 속도와 회전량이 변함
- 차체 폭을 센서가 직접 인식하지 못함

이 한계 때문에 본 프로젝트는 정밀 주행보다는, 제한된 센서로 안정적인 장애물 회피를 구현하는 데 초점을 두었습니다.

## 14. 향후 개선 방향

성능을 더 높이기 위해 추가할 수 있는 개선 사항은 다음과 같습니다.

- SG90 서보모터를 이용해 초음파 센서를 좌우로 회전
- 좌/우 초음파 센서 추가
- 포토인터럽터 센서를 추가하여 엔코더 휠 사용
- 좌우 바퀴 속도 보정
- 블루투스를 통한 실시간 튜닝값 변경
- EEPROM을 이용한 튜닝값 저장
- 모터 전원 안정화
- 더 효율적인 모터 드라이버 사용

## 15. 결론

이 프로젝트는 초음파 센서 1개와 엔코더 없는 2WD RC카 환경에서, 각도 기반 제어 대신 상태머신 기반의 거리 탐색 방식으로 장애물을 회피하도록 설계되었습니다.

핵심 아이디어는 다음과 같습니다.

```text
정확히 몇 도 회전하는 차를 만드는 것이 아니라,
갈 수 있는 공간을 찾고,
차체 폭을 고려해 조금 더 회전한 뒤,
일정 시간 과감하게 전진하는 차를 만든다.
```
